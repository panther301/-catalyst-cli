Catalyst-icon is an independent Angular library for rendering and managing custom SVG icons. You can register icons as raw SVG strings or load them asynchronously from external URLs. It gives you full control over icon sizing, colors, and styling.

Why Catalyst Icon?

  • Centralized icon registry service
  • Support for dynamic URL fetching (local assets or external raw files)
  • Angular Signals for reactive parameter updates
  • Built-in automatic DOM sanitization for secure XSS prevention
  • Flexible styling configuration via inputs and CSS

CatalystIcon API

Inputs

Name Type Default Description
icon string Required The name of the registered icon to render.
class string '' Append custom CSS classes directly to the root icon tag. Aliased as cssClass internally.
height string | number 24 Sets the height of the SVG. Numbers are treated as `px` suffix automatically.
width string | number 24 Sets the width of the SVG. Numbers are treated as `px` suffix automatically.
color string currentColor Sets the CSS color value applied to the style attribute of the inline icon. Default utilizes CSS cascading logic.

CatalystIconRegistryService API

Methods

Signature Description
registerIcon(name: string, content: string, type: 'svg' | 'url' = 'svg'): void Programatically registers a single standalone icon entity by type (defaults to SVG strings).
registerIcons(icons: {{ '{' }} name: string; svg?: string; url?: string {{ '}' }}[]): void Registers multiple SVG files and URL links collectively through an array structure.
getIcon(name: string): Observable<string | undefined> Returns an Observable hook to retrieve registered SVG markup (used internally).
